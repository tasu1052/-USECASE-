# Use case Descriptions

열 병합이 안 되어서 Extensions은 그냥 아래에 썼습니다 ㅜ 

1. **회원 가입**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 비회원은 ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형(일반/전기) 등을 입력한다. 가입 버튼을 클릭한다. | 2. 회원가입이 완료되었다는 메세지를 표시한다. |
    - 참고
        ![image](https://github.com/user-attachments/assets/f160d95a-6fa3-471b-a84c-d0a150900c7b)

        
2. **회원 탈퇴**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 회원은 회원 탈퇴 버튼을 클릭한다. | 2. 탈퇴 의사를 묻는 팝업 메세지를 표시한다. |
    | 3. 회원은 는 팝업 메세지 내 탈퇴 버튼을 클릭한다. | 4. 회원 탈퇴가 완료되었다는 메세지를 표시한다. 비회원 홈 화면으로 이동한다. |
3. **로그인**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 사용자는 ID와 비밀번호를 입력한다. 로그인 버튼을 클릭한다. | 2. 로그인이 완료되었다는 메세지를 표시한다. 홈 화면으로 이동한다. |
4. **로그아웃**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 사용자는 로그아웃 버튼을 클릭한다. | 2. 로그아웃이 완료되었다는 메세지를 표시한다. 비회원 홈 화면으로 이동한다. |
5. **대여소 등록**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 관리자는 대여소 이름, 위치(도시, 주소), 자전거 보관 가능 수량, 운영 시간 등을 입력한다. 등록 버튼을 클릭한다. | 2. 대여소 등록이 완료되었다는 메세지를 표시한다. |
6. **대여소 리스트 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1.  None | 2. 등록된 대여소 리스트를 출력한다. |
    | **Extensions**
    Step 2 이후, 관리자는 특정 대여소 항목을 선택해 삭제할 수 있다. |  |
7. **대여소 상세 정보 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 대여소의 상세 정보(대여소 이름, 위치(도시, 주소), 자전거 보관 가능 수량, 운영 시간 등)를 출력한다. |
8. **자전거 등록**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 관리자는 자전거 ID, 자전거 제품명, 유형(일반/전기), 소속 대여소, 상태(사용 가능/수리 중) 등을 입력한다. 등록 버튼을 클릭한다. | 2. 자전거 등록이 완료되었다는 메세지를 표시한다. |
9. **자전거 리스트 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 등록된 자전거 리스트를 출력한다. |
    | **Extensions**
    Step 2 이후, 관리자는 특정 자전거 항목을 선택해 삭제할 수 있다. |  |
10. **자전거 상세 정보 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 자전거의 상세 정보(자전거 ID, 자전거 제품명, 유형(일반/전기), 소속 대여소, 상태(사용 가능/수리 중) 등)를 출력한다. |
11. **대여소 검색**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. 회원은 검색창에 대여소 이름을 입력한다. 검색 버튼을 클릭한다. | 2. 회원이 검색한 이름이 포함된 대여소 리스트를 출력한다. |
12. **대여소 상세 정보 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 대여소의 상세 정보(대여소 이름, 대여소 위치, 사용 가능 자전거 목록 등)를 출력한다. |
    | **Extensions**
    Step 2 이후, 자전거가 남아 있는 경우, 회원은 즉시 대여할 수 있다.
    Step 2 이후, 자전거가 없는 경우, 회원은 예약 대기를 신청할 수 있다. |  |
13. **자전거 대여 정보 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 현재 대여 중인 자전거 리스트를 출력한다. 이 때, 대여소 이름, 대여소 위치, 자전거 ID, 자전거 제품명, 자전거 유형을 보여준다. |
    | **Extensions**
    Step 2 이후, 회원은 특정 자전거를 지정된 대여소에 반납할 수 있다. |  |
14. **자전거 예약 대기 정보 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 현재 예약 대기 중인 자전거 리스트를 출력한다. 이 때, 대여소 이름, 대여소 위치, 자전거 ID, 자전거 제품명, 자전거 유형을 보여준다. |
    | **Extensions**
    Step 2 이후, 회원은 현재 예약 대기 중인 자전거에 대해 예약 대기를 취소할 수 있다. |  |
15. **요금 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 현재 대여 중인 자전거에 대해, 대여 시간과 요금을 출력한다. |
16. **이용 내역 조회**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 과거 대여 기록을 출력한다. 디폴트는 날짜 별로 출력한다. |
    | **Extensions**
    Step 2 이후, 회원은 정렬 기준을 변경하여 대여소 별로 정렬할 수 있다.
    Step 2 이후, 회원은 과거 대여 기록 중 특정항목을 선택해 삭제할 수 있다. |  |
17. **대여 정보 통계**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 현재까지의 총 대여 정보를 출력한다. |
    | **Extensions**
    Step 2 이후, 관리자는 자전거 반납 시간을 기준으로 최신순 혹은 지역별로 선택하여 정렬할 수 있다. |  |
18. **대여 금액 및 횟수 확인 통계**
    
    
    | Actor Action | System Responce |
    | --- | --- |
    | 1. None | 2. 현재까지의 총 대여 정보에 대해, 대여 금액 및 횟수를 출력한다. |
    | **Extensions**
    Step 2 이후, 관리자는 최근 1주일, 1개월, 1년 단위로 조회 단위를 선택할 수 있다. |  |
